Avant de lancer la moindre attaque, il faut un environnement solide. Un bon lab c'est la base — sans ça, tu travailles à l'aveugle. Dans cet épisode on monte tout de zéro : les VM, le réseau, les outils.

## Le setup minimal

Tu n'as pas besoin d'un serveur dédié pour commencer. Un laptop avec **8 Go de RAM minimum** (16 Go recommandé) suffit largement.

### Hyperviseur — VirtualBox ou VMware ?

|Critère|VirtualBox|VMware Workstation|
|---|---|---|
|Prix|Gratuit|Gratuit (Player)|
|Performances|Bonnes|Meilleures|
|Snapshots|Oui|Oui|
|Compatibilité|Excellente|Excellente|
|Recommandé pour|Débutants|Intermédiaire+|

Pour débuter : **VirtualBox** — gratuit, open source, largement documenté.

📌 Téléchargement : [virtualbox.org](https://www.virtualbox.org/)

## Les VM essentielles

### 1. Kali Linux — ta machine d'attaque

Kali est la distribution de référence pour le pentest et le red team. Elle embarque plus de 600 outils préinstallés.

![Kali Linux](https://www.kali.org/images/kali-dragon-icon.svg) _Kali Linux — la distribution offensive de référence_

- **RAM recommandée** : 4 Go
- **Disque** : 40 Go minimum
- **Usage** : reconnaissance, exploitation, post-exploitation

 Téléchargement : [kali.org/get-kali](https://www.kali.org/get-kali/)

### 2. Windows 10/11 — ta cible principale

La majorité des environnements d'entreprise tournent sous Windows. Il est indispensable d'avoir une VM Windows pour :

- Comprendre l'environnement cible
- Tester des payloads avant de les déployer
- Pratiquer la post-exploitation Windows

 Téléchargement légal : [Microsoft Evaluation Center](https://www.microsoft.com/en-us/evalcenter/)

### 3. Windows Server — pour simuler un AD

Active Directory est au cœur de 90% des réseaux d'entreprise. Une VM Windows Server te permet de monter ton propre lab AD.

- **RAM recommandée** : 4 Go
- **Usage** : simulation d'environnement entreprise, attaques AD

### 4. Metasploitable / DVWA — cibles vulnérables intentionnelles

Des machines volontairement vulnérables pour s'entraîner légalement.

|Machine|Usage|
|---|---|
|[Metasploitable 2](https://sourceforge.net/projects/metasploitable/)|Cible Linux vulnérable|
|[DVWA](https://dvwa.co.uk/)|Web app vulnérable|
|[VulnHub](https://www.vulnhub.com/)|Catalogue de VMs vulnérables|

## La configuration réseau du lab

C'est le point que beaucoup négligent. Un mauvais réseau = des attaques qui sortent de ton lab et touchent ton réseau réel. À éviter absolument.

### Le mode réseau recommandé : Host-Only + NAT

```
                🌐 Internet
                     |
                  [ NAT ]
                     |
          ┌──────────┴──────────┐
          │                     │
     🖥️ Host (ton PC)     🐉 Kali VM
                               |
                     ┌─────────┴─────────┐
                     │ Host-Only Network │
                     │ (réseau isolé)    │
                     └─────────┬─────────┘
                               |
        ┌───────────────┬───────────────┐
        │               │               │
   🎯 Metasploitable   🪟 Windows    🎯 Autres cibles
```

- **NAT** : Kali accède à internet, les cibles non
- **Host-Only** : communication entre VMs uniquement, isolée du réseau réel

Dans VirtualBox :

1. `File → Host Network Manager → Create`
2. Assigne le réseau Host-Only à tes VMs cibles
3. Ajoute une carte NAT à Kali pour les updates

## Les outils indispensables

Kali les embarque déjà pour la plupart, mais voici les essentiels à connaître :

### Reconnaissance

|Outil|Usage|
|---|---|
|`nmap`|Scan de ports et services|
|`masscan`|Scan rapide sur grands périmètres|
|`theHarvester`|OSINT — emails, domaines, IPs|
|`amass`|Énumération de sous-domaines|

### Exploitation

|Outil|Usage|
|---|---|
|`metasploit`|Framework d'exploitation|
|`burpsuite`|Proxy web, attaques applicatives|
|`sqlmap`|Injection SQL automatisée|
|`hydra`|Brute force de services|

### Post-exploitation

|Outil|Usage|
|---|---|
|`mimikatz`|Extraction de credentials Windows|
|`bloodhound`|Cartographie Active Directory|
|`impacket`|Attaques protocoles Windows|
|`chisel`|Tunneling et pivoting|

## Ce qu'on retiendra

- **VirtualBox** + **Kali** + **une cible Windows** = le trio de base
- Le réseau **Host-Only + NAT** isole ton lab du réseau réel
- **TryHackMe** pour débuter, **HackTheBox** pour progresser

Dans le prochain épisode on attaque la **reconnaissance** : collecter un maximum d'informations sur une cible sans se faire repérer.
