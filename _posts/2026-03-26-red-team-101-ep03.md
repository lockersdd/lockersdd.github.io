La reconnaissance c'est la phase la plus sous-estimée du red team. La plupart des débutants veulent aller direct à l'exploitation. Les pros passent parfois **plus de temps en recon qu'en attaque**. Voici pourquoi — et comment faire.


## Avant tout — les définitions de base

Si tu débutes, certains termes peuvent sembler flous. On clarifie tout ici avant d'aller plus loin.

### C'est quoi un domaine ?

Un **domaine** c'est l'adresse principale d'un site ou d'une organisation sur internet. C'est ce que tu tapes dans la barre de ton navigateur.

```
exemple.com       ← domaine principal
google.com        ← domaine principal
hackthebox.com    ← domaine principal
```

### C'est quoi un sous-domaine ?

Un **sous-domaine** c'est une subdivision du domaine principal. Il permet à une organisation d'héberger plusieurs services sous le même nom de domaine.

```
mail.exemple.com      ← sous-domaine pour la messagerie
vpn.exemple.com       ← sous-domaine pour le VPN
admin.exemple.com     ← sous-domaine pour l'administration
dev.exemple.com       ← sous-domaine pour l'environnement de développement
```

> 💡 En red team, les sous-domaines sont une mine d'or. Les équipes IT oublient souvent de sécuriser les sous-domaines de dev, staging ou admin — et c'est souvent par là qu'on entre.

### C'est quoi une adresse IP ?

Une **adresse IP** c'est l'adresse numérique réelle d'un serveur sur internet. Le domaine est juste un nom lisible qui pointe vers cette IP.

```
exemple.com  →  93.184.216.34
```

C'est le rôle du **DNS** (Domain Name System) de faire cette traduction.

### C'est quoi le DNS ?

Le **DNS** c'est l'annuaire d'internet. Il traduit les noms de domaine en adresses IP. Sans DNS, tu devrais mémoriser des suites de chiffres pour accéder à chaque site.

```
Tu tapes : google.com
    ↓
DNS traduit : 142.250.74.46
    ↓
Ton navigateur se connecte au serveur
```

### C'est quoi un port ?

Un **port** c'est une porte d'entrée sur un serveur. Chaque service écoute sur un port différent.

```
Port 80   → HTTP  (site web non sécurisé)
Port 443  → HTTPS (site web sécurisé)
Port 22   → SSH   (accès terminal distant)
Port 21   → FTP   (transfert de fichiers)
Port 3389 → RDP   (bureau distant Windows)
```

> 💡 Scanner les ports d'une cible permet de savoir quels services tournent sur ses serveurs — et donc quelles portes sont ouvertes.

### C'est quoi l'OSINT ?

**OSINT** = Open Source INTelligence. C'est le renseignement à partir de sources **publiques et légales** : internet, réseaux sociaux, bases de données publiques, journaux, etc.

En red team, l'OSINT permet de collecter un maximum d'informations sur une cible **sans jamais la toucher directement**.

### C'est quoi un ASN ?

Un **ASN** (Autonomous System Number) est un identifiant unique attribué à un réseau géré par une organisation (entreprise, FAI, hébergeur). Il permet de trouver toutes les plages d'adresses IP appartenant à une organisation.

```
# Exemple : trouver les IPs d'une entreprise via son ASN
AS15169 → Google LLC → 8.8.8.0/24, 8.35.192.0/20...
```

## Reconnaissance passive vs active

Il existe deux grandes familles de reconnaissance :

||Passive|Active|
|---|---|---|
|**Définition**|Collecter sans toucher la cible|Interagir directement avec la cible|
|**Risque détection**|Très faible|Élevé|
|**Sources**|Internet, bases publiques|Serveurs cibles, DNS, ports|
|**Exemples**|WHOIS, Google, Shodan|Nmap, ping, traceroute|

**Règle d'or** : commencer toujours par le passif avant de passer à l'actif.

## La reconnaissance passive

### 1. WHOIS — infos sur le domaine

```bash
whois cible.com
```

Tu obtiens : registrar, dates d'enregistrement, parfois nom/email du propriétaire.

### 2. Google Dorks — chercher ce que Google a indexé

Les **Google Dorks** sont des requêtes avancées qui permettent de trouver des informations sensibles exposées accidentellement sur internet.

```
site:cible.com filetype:pdf          ← tous les PDFs indexés
site:cible.com inurl:admin           ← pages admin exposées
site:cible.com intitle:"index of"    ← répertoires ouverts
"@cible.com" filetype:xls            ← fichiers Excel avec emails
site:cible.com inurl:login           ← pages de connexion
```

 Référence : [Google Hacking Database — Exploit-DB](https://www.exploit-db.com/google-hacking-database)

### 3. Shodan — le moteur de recherche des objets connectés

**Shodan** indexe tous les services exposés sur internet. Contrairement à Google qui indexe des pages web, Shodan indexe des **serveurs, caméras, routeurs, imprimantes, systèmes industriels**.

```
hostname:cible.com
org:"Nom de l'entreprise"
ssl.cert.subject.cn:cible.com
port:3389 country:FR              ← RDP exposés en France
```

 Référence : [shodan.io](https://www.shodan.io/)

### 4. theHarvester — OSINT automatisé

```bash
theHarvester -d cible.com -b google,bing,linkedin
```

Collecte automatiquement : emails, sous-domaines, IPs, noms d'employés.

### 5. Énumération de sous-domaines

```bash
# Avec amass (le plus complet)
amass enum -passive -d cible.com

# Avec subfinder (rapide)
subfinder -d cible.com

# Via les logs de certificats SSL — zéro contact avec la cible
curl "https://crt.sh/?q=%.cible.com&output=json"
```

> 💡 **crt.sh** exploite les **Certificate Transparency Logs** — des registres publics qui enregistrent tous les certificats SSL émis. Chaque sous-domaine qui obtient un certificat HTTPS y apparaît. C'est 100% passif et très efficace.

 Référence : [crt.sh](https://crt.sh/)

### 6. LinkedIn & réseaux sociaux

Les employés publient souvent des informations utiles sans le savoir :

- Stack technique utilisé dans les offres d'emploi
- Organigramme de l'entreprise
- Certifications et outils maîtrisés
- Conférences et slides publiés

## La reconnaissance active

### 1. Scan de ports avec Nmap

```bash
# Scan des 1000 ports les plus communs
nmap -sV -sC cible.com

# Scan complet de tous les ports (discret)
nmap -sS -T2 -p- cible.com

# Détection OS
nmap -O cible.com
```

> ⚠️ Un scan nmap **laisse des traces** dans les logs du pare-feu. En engagement réel, utilise `-T1` ou `-T2` pour ralentir le scan et réduire le bruit généré.

**Les options Nmap essentielles à connaître :**

|Option|Signification|
|---|---|
|`-sV`|Détecte les versions des services|
|`-sC`|Lance les scripts par défaut|
|`-sS`|Scan SYN (discret, ne complète pas la connexion)|
|`-T1` à `-T5`|Vitesse du scan (T1 = lent/discret, T5 = rapide/bruyant)|
|`-p-`|Scanne tous les 65535 ports|
|`-O`|Tente de détecter l'OS|

### 2. Énumération DNS

```bash
# Transfert de zone (si le serveur est mal configuré)
dig axfr @ns1.cible.com cible.com

# Brute force de sous-domaines
dnsrecon -d cible.com -t brt

# Lookup simple
nslookup cible.com
dig cible.com ANY
```

> 💡 Un **transfert de zone DNS** mal configuré peut révéler toute l'arborescence des sous-domaines d'une organisation en une seule requête. C'est rare mais ça existe encore.

### 3. Fingerprinting web

```bash
# Identifier les technologies d'un site
whatweb https://cible.com

# Inspecter les headers HTTP
curl -I https://cible.com

# Nikto — scanner de vulnérabilités web basique
nikto -h https://cible.com
```

## Le workflow complet de recon

```
        [ Domaine principal ]
                 |
                 ↓
        [ Sous-domaines ]
   (amass / subfinder / crt.sh)
                 |
                 ↓
        [ IP / ASN ]
      (whois / dig / shodan)
                 |
                 ↓
        [ Ports & Services ]
        (nmap / masscan)
                 |
                 ↓
        [ Technologies ]
   (whatweb / wappalyzer / curl)
                 |
                 ↓
        [ Employés / Emails ]
   (theHarvester / LinkedIn)
                 |
                 ↓
   [ Vulnérabilités potentielles ]
                 |
                 ↓
        [ Plan d'attaque ]
```

## Ce qu'on retiendra

- Un **domaine** = adresse principale, un **sous-domaine** = service rattaché
- Le **DNS** traduit les noms en IPs, les **ports** sont les portes des serveurs
- Toujours commencer par la **recon passive** — zéro contact avec la cible
- **Google Dorks + Shodan + crt.sh** = trio de base
- **Nmap** pour la recon active — avec discrétion (`-T2`)
- **Tout noter** — une info inutile aujourd'hui peut devenir clé demain
